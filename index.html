Shortcuts

Here are some very useful shortcuts I use all the time.

Strg + Shift + D Copy and paste the current line.
Strg + Shift + G Wrap the current selection with an HTML-Tag.
Alt + . Close the preceding open tag.
Strg + K, Strg + B Show/hide the sidebar.
Strg + D Find and expand the current selection.
Strg + Shift + V Paste and indent.
Strg + Shift + Up/Down Move the current line/selection up/down.
Strg + G Jump to a specific line number.
Strg + P Go to a specific File.

Settings

These are some of the user settings I added under Preferences > Settings - User.

"bold_folder_labels" The folder labels in your sidebar are displayed in bold, making them more distinguishable from the files.
"highlight_modified_tabs" Highlight tabs that have not been saved yet.
"scroll_past_end" Enables you to scroll past the last line of your file.
"match_brackets" Match your open bracket directly with a closing one while typing.

Structure

Working with Grunt and trying to use a meaningful CSS naming and structuring methodology I came up with this folder structure.

The "src" folder is for development only, where you can find all of the SCSS code, uncompressed JavaScript and uncompressed images or placeholders you used for development.

The "dist" folder is the one you upload on a server when deploying your project. Here are all minified and compressed files and images, automatically created by my Grunt setup.

This is how my Gruntfile.js looks like.

Gruntfile.js

It does my SCSS preprocessing, hints mistakes in my JavaScript code, minifies it all and puts it in the "dist/css" and "dist/js" folders. The only thing I have to do when I start coding is to navigate to my project folder and type "Grunt" in the command line.

Grunt

These are the steps for installing Grunt when using my basic boilerplate:

npm install -g grunt -cli Install the grunt command line interface globally on your pc.
npm init Go to your project folder and create a package.json file.
npm install -S grunt Install the grunt dev dependendy.
npm install Install all dev dependencies specified in your package.json file.
